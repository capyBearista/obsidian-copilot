# User Documentation Guidelines

**Audience**: Non-technical Obsidian users
**Parent Context**: Extends [../AGENTS.md](../AGENTS.md)

## Style & Voice
- **Tone**: Helpful, empowering, and clear.
- **Focus**: Explain *what* the feature does and *how* to use it, not the technical implementation.
- **Visuals**: Reference images in the `images/` directory when explaining UI.

## Rules
- ❌ **MUST NOT** include code snippets or technical internal references.
- ✅ **DO**: Use Markdown formatting (bolding for UI elements, backticks for commands).
- ✅ **DO**: Update the index in `docs/index.md` if adding a new page.
- ✅ **DO**: Ensure the "The Why" section (privacy and ownership) is consistently represented.

## Structure
- Each major feature gets its own `.md` file.
- Use `###` for sub-sections to keep the table of contents clean.
